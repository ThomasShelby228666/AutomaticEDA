# AutoEDA - Автоматический разведочный анализ данных

Веб-приложение для автоматического разведочного анализа данных (EDA) с графическим интерфейсом на Gradio.

## Описание

AutoEDA предоставляет удобный веб-интерфейс для быстрого анализа загруженных CSV-файлов. Приложение автоматически генерирует:

- Обзорную информацию о датасете (типы данных, пропуски, уникальные значения)
- Статистическое описание признаков
- Визуализацию пропущенных значений
- Корреляционную матрицу (включая закодированные категориальные признаки)
- Гистограммы распределения числовых признаков

## Быстрый старт

### Запуск с помощью Docker

1. Клонируйте репозиторий:
```bash
git clone https://github.com/ThomasShelby228666/AutomaticEDA.git
cd AutomaticEDA
```

2. Соберите Docker-образ:
```bash
docker build -t autoeda .
```

3. Запустите контейнер:
```bash
docker run -p 7860:7860 autoeda
```

4. Откройте браузер и перейдите по адресу http://localhost:7860


### Локальный запуск без Docker

1. Установится зависимости
```bash
pip install -r requirements.txt
```

2. Запустите приложение
```bash
python app.py
```

## Технологии

- Python 3.9+
- Pandas - обработка и анализ данных
- Scikit-learn - кодирование категориальных признаков
- Matplotlib/Seaborn - визуализация данных
- Gradio - веб-интерфейс

##  Возможности

**`analyzer.py`**
```
    - Подсчет статистических показателей
    - Выявление пропущенных значений
    - Кодирование категориальных признаков
    - Расчет корреляционной матрицы
```

**`visualizer.py`**
```
    - Тепловая карта пропущенных значений
    - Корреляционная матрица с аннотациями
    - Гистограммы распределений с KDE
```

**`app.py`**
```
    - Интеграция анализа и визуализации
    - Веб-интерфейс с вкладками
    - Сохранение графиков во временные файлы
    - Обработка ошибок
```

## Конфигурация

Приложение по умолчанию запускается на порту 7860. Для изменения порта отредактируйте файл app.py:
```bash
demo.launch(server_name="0.0.0.0", server_port=7860)  # Измените порт при необходимости
```

## Требования к входным данным

- Формат файла: CSV
- Кодировка: UTF-8 (рекомендуется)
- Разделитель: запятая (стандартный CSV)
- Поддерживаются как числовые, так и категориальные признаки
